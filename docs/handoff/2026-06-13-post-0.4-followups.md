## How to work
- **Validation gate:** `npm run build` + `npm run typecheck` (3 tsconfigs) + `npm test`. Baseline = **3 pre-existing env-flaky Shiki failures** in `tests/primitives/highlighter.test.ts` — any OTHER failure is a regression. Plus `npm run test:react` (5/5) for the React wrappers.
- **Showcase:** `npm run build` then `npm run examples` → `http://localhost:8000/examples/composable/index.html` (served from repo root).
- **Storybook:** `npm run storybook` (dev, :6006) / `npm run build-storybook`. Story IDs: kebab(title)+`--`+kebab(story); docs page = `…--docs&viewMode=docs`.
- **a11y audit:** `node scripts/audit-a11y.mjs` (axe in real browser, shadow-aware, light+dark; needs the showcase served on :8000). Target: 0 kit violations.
- **Headless visual checks:** write an ephemeral `.mjs` in the repo root (so node resolves `playwright`), drive Storybook/showcase, screenshot, delete it. **Verify VISUALLY** — "it compiles" missed real bugs this session.
- **Norms:** show diffs / commit per logical step (per-phase commits fine on a working branch). **Use REST to merge PRs** (`gh api --method PUT repos/kitn-ai/chat/pulls/N/merge -f merge_method=merge`) — `gh pr merge/edit` trip the Projects-classic bug. Merging to main → release-please PR → merging THAT publishes (OIDC). SolidJS: never destructure props (use `splitProps`).
- **Subagent pattern that worked:** dispatch focused implementer agents with explicit specs + mandatory verification; for anything visual, screenshot-verify. The `cn()` helper (`src/utils/cn.ts`) registers the custom `text-caption/meta/body/title` size utilities with tailwind-merge — don't regress that.

### 2. Custom cross-platform scrollbar styling (light + dark)
Default Windows/Linux scrollbars look out of place vs the Mac overlay style. Add consistent, themed scrollbar styling to the kit's scrollable regions (message list, conversation list, dropdown `[role=menu]`, hover-card content, prompt textarea, code blocks). Use BOTH `scrollbar-width: thin` + `scrollbar-color` (Firefox) AND `::-webkit-scrollbar`/`-thumb`/`-track` (Chromium/WebKit), thin, rounded, subtle — thumb on hover slightly stronger. Theme via tokens for light/dark (reuse `--color-border`/`--color-muted-foreground` or add `--kc-scrollbar-thumb`/`--kc-scrollbar-track` with `.dark` overrides). It must apply INSIDE the shadow DOM — put the base CSS in the elements' style layer (`src/elements/styles.css` `@layer base`, which compiles to `compiled.css`) and/or `theme.css`, scoped so it doesn't leak to the host page. Don't force scrollbars where none are needed (style only, don't change overflow). Verify in Storybook + showcase, light & dark; note macOS hides overlay scrollbars by default (test by forcing `overflow:scroll` or on a non-Mac/Chromium).

## Notes / gotchas carried over
- Storybook **autodocs** clips shadow-portaled overlays (a Storybook wrapper has transform+overflow:hidden). `inline:false` "fixes" it but broke docs module loading, so it was **reverted** — leave docs as-is; overlays work in Canvas view and real apps. Don't re-apply inline:false.
- Storybook rewrites **markdown** relative links to broken `?path=` URLs — use HTML `<a>` for file links in MDX. GFM tables need `remark-gfm` (already configured in `.storybook/main.ts`). Code fences only highlight supported langs — use `html` for Vue/Svelte SFCs (not `vue`/`svelte`).
- React wrappers: `frameworks/react/index.tsx` is **generated** in postbuild (don't hand-edit); `runtime.tsx` is the hand-written generic bridge. Rebuild + commit after element API changes.
