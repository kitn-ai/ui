# Design — Storybook docs IA: friendly names, framework code-tabs, React rename, SolidJS tier (2026-06-14)

Implementation spec for a Storybook information-architecture + docs pass. Goal: make
**Web Components the primary, obvious surface**, give every element **per-framework copy-paste
code examples** (one framework shown at a time), and demote the SolidJS layer to a clearly
subordinate "advanced" tier — without losing any reference. Decided via brainstorming
2026-06-14. Builds on the generated spec system
(`docs/superpowers/specs/2026-06-13-web-component-spec-system*`) — same no-AI, no-drift,
generated-from-source philosophy.

The work splits into **two independently shippable plans** (one spec, two plans):

- **Plan 1 — `2026-06-14-storybook-friendly-naming-framework-tabs.md`**: the `displayName`
  keystone, React wrapper rename, the framework-usage generator, the framework code-tabs
  block, and retitling element stories to friendly names.
- **Plan 2 — `2026-06-14-storybook-solid-ia-regroup.md`**: regroup the SolidJS components +
  UI primitives under a single "SolidJS (advanced)" tier, collapse duplicate Solid stories,
  and tag-filter the tier out of the default sidebar.

## Why

Today the sidebar shows two near-co-equal catalogs — `Web Components/kc-*` (41 elements,
the product) and `Components/*` (~32 SolidJS pieces, ~25 of which are 1:1 duplicates of an
element) — plus `UI/*` (13 primitives). A consumer can't tell which one they want, and the
duplication doubles the surface for no benefit. Most consumers want Web Components, in
whatever framework. The SolidJS layer only matters to the rare dev authoring a custom chat
in SolidJS.

## Decisions (locked)

### Friendly names everywhere

- The **friendly name** of an element is its `className` minus the `Kc` prefix and `Element`
  suffix: `KcArtifactElement → Artifact`, `KcChainOfThoughtElement → ChainOfThought`. It is
  **generated** (no hand-maintained map) and emitted as a new `displayName` field on every
  element in `src/elements/element-meta.json`.
- **`displayName` is the single source** for: the React wrapper export name, the Solid wrapper
  export name, the Storybook sidebar label, and the API-tab lookup key.
- Element story titles change `Web Components/kc-artifact → Web Components/Artifact`. The
  literal tag `kc-artifact` appears **only inside the code-tab snippets** (HTML / Vue / Angular).

### React wrapper rename `Kc*` → bare name

- React wrappers export the bare friendly name: `KcArtifact → Artifact`, `KcArtifactProps →
  ArtifactProps`. Generated by `scripts/gen-element-react.mjs`.
- This **supersedes** the earlier "React wrappers follow tags (`KcChat`, `KcTool`)" decision
  (kitn-chat-state / kc-prefix notes). Rationale: parity with the SolidJS export names and a
  cleaner consumer API; devs who want the old name alias on import
  (`import { Artifact as KcArtifact } from '@kitn.ai/chat/react'`).
- **Breaking change for React consumers → minor bump pre-1.0** (release-please
  `bump-minor-pre-major`). Sweep `examples/react`, docs, and `llms.txt` snippets.
- **One known collision:** `KcImage → Image` shadows the browser global `Image` constructor
  in any file that imports it. Acceptable — devs alias it. It is the only name in the set
  that collides with a runtime global (`Source`, `Embed`, `Form`, `Card`, `Context` are all
  fine identifiers).

### Framework code-tabs (one framework at a time)

- Each element's **API tab** gains a tabbed code block: **HTML · React · Vue · Angular ·
  Solid**, one shown at a time. Built as a small **React** component inside `.storybook/api-tab.tsx`
  (the API tab is a Storybook manager addon = React; the Solid `UI/*` primitives are in the
  preview iframe and cannot be reused here — and don't need to be).
- Snippets are **generated** by a new `scripts/gen-framework-usage.mjs` into
  `src/elements/framework-usage.json`, keyed by tag, run inside `build:api`. No drift.
- The **Solid tab is shown only when a same-named Solid export exists** (i.e. a
  `component-meta.json` entry whose name equals the element's `displayName`). Elements with no
  Solid twin (e.g. `kc-chat`) show four tabs.
- The active framework is **remembered across elements within a session** (module-level
  `let`, seeded into `useState`) so a React dev picks "React" once.

### Snippet rules (per framework, deterministic)

For each element the snippet includes its **required props** (`optional === false`) bound, plus
**all events** wired. Scalar props bind as attributes; non-scalar props bind as JS properties.

| Framework | Tag/import | Prop binding | Event binding |
|---|---|---|---|
| HTML | `<kc-artifact>` + `import '@kitn.ai/chat/elements'` | scalar → attribute; non-scalar → `el.files = …` in `<script>` | `el.addEventListener('navigate', e => …)` |
| Vue | `<kc-artifact>` | scalar → `:files="…"`; non-scalar → `:files.prop="…"` | `@navigate="onNavigate"` |
| Angular | `<kc-artifact>` | `[files]="files"` | `(navigate)="onNavigate($event)"` |
| React | `import { Artifact } from '@kitn.ai/chat/react'` | `files={files}` | `onNavigate={e => …}` (event name → `onName()`) |
| Solid | `import { Artifact } from '@kitn.ai/chat'` | `files={files}` | `onNavigate={e => …}` (same `onName()` form) |

The React/Solid columns read almost identically — the intended consistency. The Solid snippet
is an approximation built from the element's props/events (not the Solid component's exact
signature); this is acceptable for a usage example and noted in the generated file's header.

### IA: three tiers, one primary

```
Docs/
Web Components/                 ← primary, default view (friendly names)
SolidJS (advanced)/
  Components/                   ← the Solid-only composed pieces + collapsed duplicates
  Primitives/                   ← the 13 base UI building blocks
Examples/  Patterns/
```

- Every SolidJS story (Components + UI) is retitled under `SolidJS (advanced)/` and tagged
  `['solid']`. The default sidebar hides the `solid` tag; a SolidJS author flips it on.
- **UI primitives stay grouped as primitives** under `SolidJS (advanced)/Primitives/` — they
  are the base layer the components compose from; the landing note says so.

### Duplicate Solid stories: collapse, don't delete

The ~25 `Components/*` stories that duplicate a `kc-*` element are **collapsed** (retitled
under `SolidJS (advanced)/Components/`, reduced to a single representative story, tagged
`solid`), **not deleted**. Rationale:

- The generated **"Composed from"** links on each element's API tab point at
  `Components/<Name>` story ids — hard-deleting the story 404s those links.
- The Solid component's API-tab spec is only **reachable via its story** (the API tab attaches
  to story ids). No story → no way to view the Solid component's reference.

Collapsing keeps links + reference intact while removing the sidebar competition. (A harder
prune that accepts losing those pages is a possible future call, not this pass.)

The **Solid-only** components keep their fuller stories (still retitled + tagged): no `kc-*`
twin exists for `conversation-item`, `chat-container`, `scroll-button`, `slash-command`,
`message-narrow` (`context`, `prompt-suggestion` are borderline — keep only if the Solid
component genuinely differs from its near-twin element).

## Files touched

**Plan 1**
- `scripts/_ts-helpers.mjs` — add shared `displayName(className)` helper.
- `scripts/gen-element-api.mjs` — emit `displayName` per element into `element-meta.json`.
- `src/elements/element-meta.json` — regenerated (+`displayName`).

- `src/elements/framework-usage.json` — **new**, generated.
- `.storybook/api-tab.tsx` — WC lookup by `displayName`; add `FrameworkTabs` block to `ElementPanel`.

- `examples/react/**`, `docs/**`, `scripts/gen-llms.mjs` — React import-name sweep.
- `package.json` — wire `gen-framework-usage.mjs` into `build:api`.
- Tests under `tests/` for the helper, the framework-usage generator (artifact case), the displayName emit.

**Plan 2**
- `src/components/*.stories.tsx`, `src/ui/*.stories.tsx` — retitle under `SolidJS (advanced)/…`, collapse duplicates, add `tags: ['solid']`.
- `.storybook/api-tab.tsx` — component lookup by title suffix + `match`/`matchesComponentStory` for new story-id prefixes.
- `.storybook/manager.ts` (or `preview.ts`) — default-hide the `solid` tag (exact SB 10.4 filter API confirmed against context7 docs during execution).
- A `SolidJS (advanced)` landing MDX (intro + "you probably want Web Components" signpost).
