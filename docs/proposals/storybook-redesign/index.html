02 When to use

Show an AI-generated artifact — a web page, doc, image or PDF your backend hosts — beside the conversation, typically inside a kai-resizable panel. The component frames content; your backend serves it, so relative links, back/forward, reload and multi-format work natively.

03 How to use

  1. Register once. import '@kitn.ai/chat/elements' anywhere before the element mounts.
  2. Point it at your artifact. Set src to the hosted URL.
  3. Feed the Code tab. Set the files property — a JS array of { path, url?, code?, language?, type? }.
  4. Listen. kai-navigate, kai-tab-change and kai-file-select tell you what the user did.
html
<!-- register, then drive via properties -->
<kai-artifact id="art" src="/artifacts/site/index.html"></kai-artifact>

<script type="module">
  import '@kitn.ai/chat/elements';
  const art = document.getElementById('art');
  art.files = [{ path: 'index.html', language: 'html' }];
  art.addEventListener('kai-navigate', (e) => console.log(e.detail.url));
</script>
Sandbox is locked down by default — allow-scripts allow-forms, not allow-same-origin. Widen it via the sandbox attribute only if you trust the artifact.
