<Aside type="tip" title="When to use">
Reach for `<kc-message>` when you want to build your own message list — a custom layout, a non-Solid app, or anywhere you need to control how the thread is iterated. If you want a full chat surface in one tag (thread + input + suggestions), use `<kc-chat>` instead. In SolidJS, compose the `Message`, `MessageContent`, and `MessageActionBar` primitives directly for finer control.
</Aside>

## Usage

Register the element once, then set the `message` **JS property** and listen for `kc-message-action`:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-message id="msg" style="display:block;"></kc-message>

<script type="module">
  const msg = document.getElementById('msg');

  // `message` is a JS property — not an attribute
  msg.message = {
    id: 'm-1', role: 'assistant',
    content: "Here's the plan:\n\n```js\nconst kit = useChat();\n```",
    reasoning: { text: 'The user wants X, so I should do Y.' },
    tools: [{ type: 'search', state: 'output-available', output: { hits: 3 } }],
    actions: ['copy', 'like', 'dislike', 'regenerate'],
  };

  msg.addEventListener('kc-message-action', (e) => {
    console.log(e.detail.action, e.detail.messageId);
  });
</script>
```

A few things worth knowing:

- **`message` is a JS property.** Assign it with `el.message = { … }`. For simple cases where you only need role and text, use the scalar `role` + `content` attributes (or properties) instead of a full object.
- **Markdown** is on by default for `role="assistant"` and off for `role="user"`. Override with the `markdown` boolean attribute when you need the opposite.
- **Actions** live in `message.actions` as an array of built-in names (`'copy'`, `'like'`, `'dislike'`, `'regenerate'`, `'edit'`) and/or custom `{ id, label, icon?, tooltip? }` descriptors. The element merges them; every click fires `kc-message-action`. Alternatively, place `<kc-action>` children in the element's light DOM and skip the JS array entirely — handy for plain HTML.
- **`actionsReveal`** controls when the action bar appears: `'always'` (default) keeps it visible at all times; `'hover'` hides it until the user hovers the row.
- **Avatars** can come from `message.avatar` (the object), or from the convenience scalar props `avatar-src` / `avatar-fallback` for simple one-off cases.

To import via npm instead of the CDN, use:

```js
import '@kitn.ai/chat/elements';
```
