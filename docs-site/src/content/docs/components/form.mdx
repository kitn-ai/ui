## Preview

<Playground client:only="solid" tag="kc-form" />

<Aside type="tip" title="When to use">
Use `<kc-form>` when an agent needs structured input before continuing ‚Äî onboarding questionnaires, task parameters, feedback forms, or any agentic step that gates on user data. The schema is the single source of truth for both UI layout and validation.
</Aside>

## Usage

Assign the form definition as a **JS property** ‚Äî not an HTML attribute:

```html
<kc-form id="my-form"></kc-form>

<script type="module">
  import '@kitn.ai/chat/elements';

  const form = document.querySelector('#my-form');

  form.data = {
    type: 'object',
    title: 'How did we do?',
    required: ['rating'],
    'x-kc-submitLabel': 'Send feedback',
    properties: {
      rating: { type: 'integer', title: 'Overall rating', minimum: 1, maximum: 5, 'x-kc-widget': 'rating' },
      comments: { type: 'string', title: 'Comments', 'x-kc-widget': 'textarea' },
    },
  };

  form.addEventListener('kc-card', (e) => {
    if (e.detail.kind === 'submit') console.log('submission data', e.detail.data);
    if (e.detail.kind === 'action') console.log('secondary action', e.detail.action);
  });
</script>
```

- **Schema ‚Üí widgets** deterministically: `string` ‚Üí text; `string + enum` ‚Üí radio/select; `integer + x-kc-widget: 'rating'` ‚Üí stars; `boolean` ‚Üí switch; `array` ‚Üí repeater/checkbox group; nested `object` ‚Üí fieldset. See the **Every widget** example for the full mapping.
- **`x-kc-*` hints** in the schema control field order (`x-kc-order`), submit label (`x-kc-submitLabel`), secondary buttons (`x-kc-actions`), placeholder (`x-kc-placeholder`), and dismissibility (`x-kc-dismissible`).
- **Events use the Card contract** ‚Äî `kc-form` fires no events of its own; it emits a single bubbling `kc-card` CustomEvent. Relevant kinds: `submit` (`detail.data` is the validated object), `action`, `ready`, `dismiss`, and `error`.
- **`resolution`** ‚Äî set `el.resolution = { kind: 'submit', data: { ‚Ä¶ } }` to swap the form for a read-only `<dl>` summary.
