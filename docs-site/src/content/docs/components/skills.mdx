## Preview

This is the live `<kc-skills>` web component, mounted in a Solid island. The element has no scalar controls — set the `skills` property array to drive what badges render.

<Aside type="tip" title="When to use">
Place `<kc-skills>` beneath an assistant message row to annotate which skills or tools were active during that turn. It fits naturally in a message footer alongside action buttons. Use the JS property API when skill data arrives from your app state; use the declarative `<kc-skill>` children API when you want to stamp badges straight from server-rendered HTML without any JavaScript wiring.
</Aside>

## Usage

Register the element once, then feed it data via the `skills` **JS property** — you cannot set an array as an HTML attribute, so assignment must happen after the element upgrades:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-skills id="skills"></kc-skills>

<script type="module">
  import '@kitn.ai/chat/elements';

  const el = document.getElementById('skills');
  el.skills = [
    { id: 'web-search', name: 'Web Search' },
    { id: 'code',       name: 'Code' },
    { id: 'memory',     name: 'Memory' },
  ];
</script>
```

Each entry in the `skills` array is `{ id: string; name: string }`. The `id` is the stable identifier (used internally for deduplication); `name` is the human-readable badge label.

**Declarative children** — as an alternative to the property API, compose `<kc-skill>` child elements. These are light-DOM data carriers that the Shadow DOM hides from view; the text content becomes the badge label and the optional `id` attribute becomes the stable identifier (falls back to the label when absent):

```html
<kc-skills>
  <kc-skill id="web-search">Web Search</kc-skill>
  <kc-skill id="code">Code</kc-skill>
  <kc-skill id="memory">Memory</kc-skill>
</kc-skills>

<script type="module">
  import '@kitn.ai/chat/elements';
</script>
```

When you provide **both** the `skills` property and `<kc-skill>` children, the prop items render first and the declarative children are appended after. This makes the declarative path a great way to add static "always-on" badges while your app streams dynamic ones in via the property.

**In SolidJS** — import the `MessageSkills` primitive directly instead of the custom element:

```tsx
import { MessageSkills } from '@kitn.ai/chat';

<MessageSkills skills={[{ id: 'web-search', name: 'Web Search' }, { id: 'code', name: 'Code' }]} />
```

### Declarative composition

The `<kc-skill>` children API provides an HTML-first alternative: stamp badges directly in markup without any JS array assignment. Each `<kc-skill>` child is a light-DOM data carrier hidden by the Shadow DOM — its text content is the badge label and its optional `id` attribute is the stable identifier.

```html
<kc-skills>
  <kc-skill id="web-search">Web Search</kc-skill>
  <kc-skill id="code">Code</kc-skill>
  <kc-skill id="memory">Memory</kc-skill>
</kc-skills>
<script type="module">import '@kitn.ai/chat/elements';</script>
```
