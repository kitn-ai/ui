## Preview

Toggle `stoppable` to reveal the stop button, and edit `text` and `stop-label` to match your copy. Click Stop in the preview to see `kc-stop` appear in the Console.

<Playground client:only="solid" tag="kc-thinking-bar" />

<Aside type="tip" title="When to use">
Place `<kc-thinking-bar>` in your message thread while the assistant is generating a response — it sits naturally above the streaming reply. Enable `stoppable` whenever the user should be able to cut a long reasoning step short; it fires `kc-stop` (no payload) so your handler can cancel the in-flight request and instruct the model to answer immediately.
</Aside>

## Usage

Register the custom elements once, then drop `<kc-thinking-bar>` into your markup:

```html
<script type="module">
  import '@kitn.ai/chat/elements';
</script>

<kc-thinking-bar text="Thinking"></kc-thinking-bar>
```

A few things worth knowing:

- **All three props are attributes.** `text`, `stoppable`, and `stop-label` can all be set as HTML attributes — no JS property assignment needed. Set `stoppable` as a bare boolean attribute (no value) to opt in to the stop button.
- **`kc-chat` does not surface this element.** `<kc-thinking-bar>` is a standalone primitive you compose yourself — it gives you full control over when it appears and disappears.
- **Customising copy** — override `text` for the shimmering label (e.g. `"Reasoning…"`) and `stop-label` for the button (e.g. `"Answer now"`).
- **Listening for `kc-stop`** — add an event listener for the `kc-stop` CustomEvent (no detail payload). This is where you cancel the in-flight request. See the **Events** table below and the **Code** panel in the Preview above for per-framework handler syntax.

### Default

The bare indicator with default copy. It pulses continuously; remove it from the DOM once the assistant reply arrives.

<Example client:visible tag="kc-thinking-bar" config={{ text: 'Thinking' }} />

### Custom label

Override `text` with something more specific to your context — useful when the assistant is doing a named task like web search or code analysis.

<Example client:visible tag="kc-thinking-bar" config={{ text: 'Searching the web…' }} />

### Stoppable

Add `stoppable` to show the stop affordance. Clicking it fires `kc-stop` (no payload) — your handler cancels the request and prompts the model to answer with what it has so far.

<Example client:visible tag="kc-thinking-bar" config={{ text: 'Reasoning', stoppable: true, stopLabel: 'Answer now' }} />

### Custom stop label

Change `stop-label` to match your product's language — "Skip reasoning", "Show answer", or anything else that makes sense in context.

<Example client:visible tag="kc-thinking-bar" config={{ text: 'Deep thinking…', stoppable: true, stopLabel: 'Skip reasoning' }} />
