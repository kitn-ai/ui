<Aside type="tip" title="When to use">
Use `<kc-confirm>` when the model proposes an action the user must explicitly approve — deploying, deleting, sending. `tone: 'warning'` signals caution; `tone: 'danger'` signals destructive intent via colour **and** icon. For equally-weighted choices that aren't approval-gated, use `<kc-choice>` instead.
</Aside>

## Usage

Set `data` in JavaScript (it's an object) and listen for the `kc-card` CustomEvent:

```html
<kc-confirm heading="Run database migration?"></kc-confirm>

<script type="module">
  import '@kitn.ai/chat/elements';
  const el = document.querySelector('kc-confirm');

  el.data = {
    body: 'This will apply 3 pending migrations to production. This cannot be undone.',
    tone: 'warning',
    actions: [
      { id: 'approve', label: 'Run migration', style: 'primary', default: true },
      { id: 'reject',  label: 'Cancel' },
    ],
  };

  el.addEventListener('kc-card', (e) => {
    const ev = e.detail; // { kind: 'action', cardId, action, payload? }
    if (ev.kind === 'action') console.log('chose', ev.action);
  });
</script>
```

- **`data`** — shape: `ConfirmCardData` from `@kitn.ai/chat`. Malformed payloads render an inline error.
- **Action styles** — `'primary' | 'default' | 'destructive'`. Mark one `default: true` to make it the keyboard default; combine with `autofocus` to focus it on mount.
- **Restore resolved state** via `el.resolution = { kind: 'action', action: 'approve' }` — useful when replaying past conversations.

## Examples

### Approve or Reject

The canonical two-action pattern: `warning` tone, primary confirm, plain cancel.

<Example client:visible tag="kc-confirm" config={{ heading: 'Run database migration?' }} />

### Destructive Action

`tone: 'danger'` with `style: 'destructive'` — colour and icon both signal severity.

<Example client:visible tag="kc-confirm" config={{ heading: 'Delete files?' }} data="destructive" />

### Choice Set

More than two actions work naturally — each becomes a button.

<Example client:visible tag="kc-confirm" data="choiceSet" />

### Dismissible

`dismissible: true` adds a close (×) affordance that fires `{ kind: 'dismiss', cardId }` — the card stays in the DOM.

<Example client:visible tag="kc-confirm" config={{ heading: 'Send email?' }} data="dismissible" />

### Resolved (Read-only)

Set the `resolution` property to render the chromed read-only view for a replayed conversation.

<Example client:visible tag="kc-confirm" config={{ heading: 'Run database migration?' }} data="resolved" />

### Error State

An empty `actions` array renders the inline error view and fires `{ kind: 'error', cardId, message }`.

<Example client:visible tag="kc-confirm" data="errorState" />
