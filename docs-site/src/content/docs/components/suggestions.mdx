<Aside type="tip" title="When to use">
Place `<kc-suggestions>` above the prompt input when the thread is empty — a fast welcome state that shows users what the assistant can do. It also works below the last assistant reply as follow-up chips. Pass the `block` flag when you want suggestions to stack as full-width rows (e.g. inside a narrow sidebar panel) rather than wrapping pills. Set `highlight` when you are rendering it as a filtered autocomplete list — the matched substring is visually emphasised in every chip.
</Aside>

## Usage

Register the custom elements once, then set the `suggestions` **property** (a JS array) and listen for `kc-select`:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-suggestions id="suggs" variant="outline"></kc-suggestions>

<script type="module">
  const suggs = document.getElementById('suggs');
  suggs.suggestions = ['Explain the architecture', 'Show me a code example'];
  suggs.addEventListener('kc-select', (e) => console.log(e.detail.value));
</script>
```

A few things worth knowing:

- **`suggestions` is a JS property**, not an attribute. Assign an array: `el.suggestions = ['…']`. Each item can be a plain string (the displayed label is also the emitted value) or a `{ label, value }` object when the two differ.
- **`variant`** is a plain HTML attribute: `outline` (default — bordered pill), `ghost` (no border, subtle hover), or `default` (filled). It controls the chip style across all items.
- **`block`** is a boolean attribute/property. When set, chips become full-width left-aligned rows instead of wrapping pills — useful in narrow containers.
- **`highlight`** is a string attribute. Pass the current search query and the matching substring is visually emphasised inside every chip label. This implicitly switches the chips to the list-row layout.
- The element emits a single `kc-select` CustomEvent whose `detail.value` is the clicked chip's value (the string itself, or `item.value` from the `{ label, value }` form). See the **Events** table below for the full payload, and the **Code** panel in the Preview for the exact listener syntax in your framework.

**Declarative children** — you can also declare chips as `<kc-suggestion>` child elements instead of setting the `suggestions` property. The `value` attribute sets the emitted value; `textContent` is the displayed label. They are light-DOM data carriers hidden by the Shadow DOM and can be mixed with the `suggestions` property (prop items render first, declarative children after):

```html
<kc-suggestions variant="outline">
  <kc-suggestion value="explain">Explain the architecture</kc-suggestion>
  <kc-suggestion value="example">Show me a code example</kc-suggestion>
</kc-suggestions>
```

### Default (outline pills)

The out-of-the-box wrapping pill row with `outline` chips. Click any chip to see the `kc-select` event fire.

<Example client:visible tag="kc-suggestions" config={{ variant: 'outline' }} />

### Ghost variant

Use `variant="ghost"` for a lighter appearance — no border, just a subtle hover state. Good when you want chips to blend into the surrounding UI rather than stand out.

<Example client:visible tag="kc-suggestions" config={{ variant: 'ghost' }} />

### Block layout

Add the `block` flag to stack suggestions as full-width left-aligned rows. Handy in narrower layouts like sidebars or constrained panels where wrapping pills would look awkward.

<Example client:visible tag="kc-suggestions" config={{ variant: 'outline', block: true }} />

### With highlighted search

Set `highlight` to the current query string and the matching substring is emphasised inside each chip — ideal for a filtered autocomplete or search-suggestion list. The `highlight` attribute also implicitly switches to the list-row layout.

<Example client:visible tag="kc-suggestions" config={{ highlight: 'Solid' }} data="search" />

### Small chips

Pass `size="sm"` for a tighter chip height — useful when suggestions appear inline alongside other compact UI, or when you just want the visual weight reduced.

<Example client:visible tag="kc-suggestions" config={{ variant: 'outline', size: 'sm' }} />

### Label and value differ

When the displayed text and the emitted value should differ, pass `{ label, value }` objects instead of plain strings. The label is shown in the chip; the value is what `kc-select` emits.

<Example client:visible tag="kc-suggestions" config={{ variant: 'outline' }} data="labeled" />
