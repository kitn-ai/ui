Reach for `<kc-scope-picker>` when your interface has multiple authors or content tags and you want users to narrow the chat or search to a subset. ## Usage

Register the custom elements once, then set `availableAuthors` and `availableTags` as **JS properties** and listen for `kc-scope-change`:

```html
<script type="module">
  import '@kitn.ai/chat/elements';
</script>

<kc-scope-picker id="scope"></kc-scope-picker>

<script type="module">
  const scope = document.getElementById('scope');

  // Both are JS properties â€” assign arrays directly, not attributes.
  scope.availableAuthors = ['Alex Chen', 'Jordan Lee'];
  scope.availableTags    = ['design', 'engineering'];

  // filters is undefined when "All Content" is selected.
  scope.addEventListener('kc-scope-change', (e) => {
    const { filters } = e.detail;
    console.log(filters ?? 'all');
  });
</script>
```

A few things worth knowing:

- **`availableAuthors` and `availableTags` are JS properties**, not attributes. Assign them with `el.availableAuthors = [...]` â€” these are arrays, so attribute assignment will not work.
- **`current-label`** is a plain HTML attribute (defaults to `'All Content'`). It controls the text shown on the trigger button. Update it after a scope is selected to reflect the active filter.
- **`kc-scope-change`** is the only event. `detail.filters` is `undefined` when the user picks "All Content", or an object `{ authors?: string[], tags?: string[] }` when they pick a specific filter. See the **Events** table below for the full payload shape.
- The dropdown renders an **Authors** section and a **Tags** section only when those arrays are non-empty â€” passing an empty array hides that section entirely.
- A CDN alternative to the module import: `<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>`.

The **Code** panel in the Preview above generates the exact per-framework snippet as you adjust the controls.

## Examples

Each example is live â€” expand **Code** to copy the snippet for your framework.

### Authors and tags

Both sections populated: the dropdown shows an **Authors** group and a **Tags** group below "All Content". Selecting any item fires `kc-scope-change` with `detail.filters`.

<Example client:visible tag="kc-scope-picker" />

### Authors only

Pass a non-empty `availableAuthors` and an empty `availableTags` to render only the Authors section. Useful when your content isn't tagged but is attributable to specific people.

<Example client:visible tag="kc-scope-picker" data="authorsOnly" />

### Tags only

Pass a non-empty `availableTags` and an empty `availableAuthors` to show only the Tags section. Common for topic-scoped knowledge bases where author attribution isn't relevant.

<Example client:visible tag="kc-scope-picker" data="tagsOnly" />

### Minimal (no filters)

When both arrays are empty, the dropdown contains only "All Content". The trigger still renders and works â€” handy while filter options are loading asynchronously.

<Example client:visible tag="kc-scope-picker" data="minimal" />

### Custom trigger label

Set `current-label` to reflect an active scope â€” for example after the user has already picked "Jordan Lee". Your app owns this state and should update the attribute whenever `kc-scope-change` fires.

<Example client:visible tag="kc-scope-picker" config={{ currentLabel: 'Jordan Lee' }} data="customLabel" />
