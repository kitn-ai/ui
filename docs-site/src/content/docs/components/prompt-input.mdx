<Aside type="tip" title="When to use">
Reach for `<kc-prompt-input>` whenever you need a chat composer outside of SolidJS — in React, Vue, Svelte, Angular, or plain HTML — and want zero style bleed in either direction. It handles its own input state uncontrolled by default, so you only need to listen for `kc-submit` to get the typed text and any staged attachments. Pin it to the bottom of a chat surface at full width for the canonical layout.
</Aside>

## Usage

Register the custom elements once, then interact via JS properties and DOM events:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-prompt-input id="input" style="display:block; width:100%;"></kc-prompt-input>

<script type="module">
  const input = document.getElementById('input');

  // Non-scalar props must be set as JS properties (not HTML attributes)
  input.suggestions = ['Summarize this thread', 'Draft a reply'];
  input.slashCommands = [
    { id: 'summarize', label: '/summarize', description: 'Summarize', category: 'Actions' },
  ];

  // Listen for events
  input.addEventListener('kc-submit', (e) => console.log('send:', e.detail.value, e.detail.attachments));
  input.addEventListener('kc-value-change', (e) => console.log('typing:', e.detail.value));
</script>
```

A few things worth knowing:

- **Registration** — prefer `import '@kitn.ai/chat/elements'` in a module bundler; the CDN `<script>` above works for plain HTML.
- **Non-scalar props** — `suggestions`, `slashCommands`, `slashActiveIds`, and `attachments` are JS arrays; assign them as properties (`el.suggestions = [...]`), never as HTML attributes.
- **Scalar props / flags** — `placeholder`, `disabled`, `loading`, `search`, `voice`, `stoppable`, `slashCompact`, and `suggestionMode` are plain attributes you can also toggle as element properties.
- **Uncontrolled by default** — leave `value` unset and the element manages its own text. Set `value` and update it on every `kc-value-change` for fully controlled behavior.
- **Loading state** — set `loading` while waiting for a reply; the send button disables. Combine with `stoppable` to swap the send button for a Stop button; clicking it fires `kc-stop`.
- **Custom toolbar buttons** — place `<kc-action id icon tooltip>` children inside the element. They are invisible data carriers (Shadow DOM hides them); the element reads them with `querySelectorAll` and renders each as a ghost icon button. Clicks fire `kc-toolbar-action` with `detail.action` = the action id.
- **Declarative slash commands** — place `<kc-slash-command command="id" description="…">Label</kc-slash-command>` children as an alternative (or supplement) to the `slashCommands` property. Prop items appear first; declarative children are appended after. Typing `/` opens the palette; selecting an entry fires `kc-slash-select`.

See the **Events** table below for every event payload, and the **Code** panel in each example for the exact per-framework snippet.

### Default

The out-of-the-box composer with starter suggestions. Click a chip or type and press Enter to fire `kc-submit`; watch the Console below.

<Example client:visible tag="kc-prompt-input" data="withSuggestions" />

### With Voice and Search

Add the `search` and `voice` boolean flags to show a Globe and Mic button in the left toolbar. Clicking either fires `kc-search` or `kc-voice` respectively — handle them to trigger web search or start voice input.

<Example client:visible tag="kc-prompt-input" config={{ search: true, voice: true }} data="withSuggestions" />

### Slash-Command Palette

Set the `slashCommands` property to enable the command palette. Typing `/` opens it; pressing Enter or clicking an entry fires `kc-slash-select` with `detail.command`. Use `slashActiveIds` to highlight currently-active commands.

<Example client:visible tag="kc-prompt-input" data="withSlashCommands" />

### Pre-Seeded Attachments

Set the `attachments` property to pre-populate staged files before the user types. The element then manages its own attachment state — the paperclip button adds more, and each chip has a remove button.

<Example client:visible tag="kc-prompt-input" data="withAttachments" />

### Loading State

Set `loading` to block submit while a response streams. Add `stoppable` to replace the send button with a Stop button — clicking it fires `kc-stop` so you can cancel the in-flight request.

<Example client:visible tag="kc-prompt-input" config={{ loading: true, stoppable: true }} data="withSuggestions" />

### Disabled

Set `disabled` to make the input entirely non-interactive — useful when the surface is in a read-only or locked state.

<Example client:visible tag="kc-prompt-input" config={{ disabled: true }} data="withSuggestions" />
