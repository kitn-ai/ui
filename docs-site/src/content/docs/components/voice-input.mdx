<Aside type="tip" title="When to use">
Reach for `<kc-voice-input>` to add mic dictation to a prompt input, search bar, or form field in any framework. In SolidJS, compose `VoiceInput` from `@kitn.ai/chat` directly for tighter reactive control.
</Aside>

## Usage

Assign the `transcribe` **function property** and listen for `kc-transcription`:

```html
<kc-voice-input id="voice"></kc-voice-input>

<script type="module">
  import '@kitn.ai/chat/elements';

  const voice = document.getElementById('voice');

  voice.transcribe = async (blob) => {
    const form = new FormData();
    form.append('audio', blob, 'recording.webm');
    const res = await fetch('/api/transcribe', { method: 'POST', body: form });
    return (await res.json()).text;
  };

  voice.addEventListener('kc-transcription', (e) => {
    document.getElementById('prompt').value = e.detail.text;
  });
</script>
```

- **`transcribe`** is a JS property (not an attribute) — assign `async (blob: Blob) => Promise<string>` after the element is in the DOM.
- To handle audio yourself, skip `transcribe` and listen for **`kc-audio-captured`** instead; it fires `{ blob }` immediately after recording stops.
- **`disabled`** prevents interaction while the assistant is streaming or mic access is pending.
