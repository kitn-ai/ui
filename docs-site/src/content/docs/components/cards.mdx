<Aside type="tip" title="When to use">
Reach for `<kc-cards>` whenever your AI response includes structured interactions — a deployment confirmation, a multi-step plan the user can approve, a list of options to pick from. Pass the server's card envelopes straight into `el.cards`; wire up `el.policy` to handle the results; and the element takes care of rendering, routing, and resolved (read-only) states.
</Aside>

## Usage

Register the element once, then set its three JS properties imperatively — none of them work as HTML attributes:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-cards id="cards"></kc-cards>

<script type="module">
  const el = document.getElementById('cards');

  // 1. The envelope stream — each item is { type, id, data, title?, resolution? }
  el.cards = [
    { type: 'confirm', id: 'deploy', title: 'Deploy to production?',
      data: { body: 'Apply 3 migrations?', tone: 'warning',
              actions: [{ id: 'go', label: 'Deploy', style: 'primary', default: true },
                        { id: 'no', label: 'Cancel' }] } },
  ];

  // 2. How the host handles what the user does
  el.policy = {
    onAction: (cardId, action, payload) => console.log('action', cardId, action, payload),
    onSubmit: (cardId, data) => console.log('submit', cardId, data),
    onOpen:   (url, target) => window.open(url, target === 'tab' ? '_blank' : '_self'),
    onError:  (cardId, message) => console.warn('card error', cardId, message),
  };
</script>
```

A few things worth knowing:

**`cards` is a JS property.** Assign it with `el.cards = [...]` after the element upgrades. Each item is a `CardEnvelope`: `{ type, id, data, title?, resolution? }`. The `type` string selects which built-in child element renders the envelope (see the built-in registry below).

**`policy` is a JS property.** Set `el.policy = { onAction, onSubmit, onOpen, onDismiss, … }` to route child card events. You only need to provide the handlers your app cares about — the rest are silently ignored. The policy is read at event time, so you can set or swap it after the element is in the DOM. See the `CardPolicy` interface in `@kitn.ai/chat` for the full shape.

**`types` is a JS property.** Pass `el.types = { 'my-type': 'my-card-element' }` to register custom type strings that map to your own `<kc-*>` child elements. Your map is merged over the built-ins, so you can also override a built-in type.

**Card events** — child cards emit a `kc-card` custom event that bubbles up to `<kc-cards>`. The element intercepts each event and calls the matching `policy` handler (`onAction`, `onSubmit`, `onOpen`, `onSendPrompt`, `onState`, `onDismiss`, `onError`). The raw event keeps bubbling past `<kc-cards>` (composed: true), so a document-level listener still works if you prefer it.

**Unknown types** — if an envelope's `type` has no registered tag, the element renders a `CardFallback` chip and fires `policy.onError` with a descriptive message. This lets your stream degrade gracefully when the server sends a card type the current client version doesn't know yet.

**Resolved cards** — set `resolution` on an envelope to render the card in its read-only chromed state (e.g. after the user has already acted). The child element receives `{ kind: 'action', action, payload? }` or `{ kind: 'submit', data }` and renders accordingly.
