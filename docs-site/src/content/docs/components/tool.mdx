Drop one after each tool invocation in an assistant message to give users a transparent look at what the agent did.</p>

<Aside type="tip" title="When to use">
Reach for `<kc-tool>` inside an assistant message row — one element per tool call, placed below the text content. It is ideal for any framework that is not SolidJS; in SolidJS, compose the `Tool` primitive directly from `@kitn.ai/chat`. The panel renders as a block sized to its content, so no extra sizing is needed.
</Aside>

## Usage

Register the custom elements once with `import '@kitn.ai/chat/elements'` (or load the CDN bundle as a secondary option), then set the call via the `tool` **JS property**:

```html
<kc-tool id="tool" open></kc-tool>

<script type="module">
  import '@kitn.ai/chat/elements';

  const tool = document.getElementById('tool');
  tool.tool = {
    type: 'database_query',
    state: 'output-available',
    input: { table: 'users', limit: 10 },
    output: { rows: 10, ms: 42 },
  };
</script>
```

A few things worth knowing:

- **`tool` is a JS property**, not an attribute. Assign it with `el.tool = { … }` — the full `ToolPart` shape is below.
- **`state`** drives the colour-coded badge: `'input-streaming'` (blue) while arguments are being assembled, `'input-available'` (amber) while the call is running, `'output-available'` (green) on success, and `'output-error'` (red) on failure.
- **`open`** is a boolean attribute/property. Set it to start the panel expanded; otherwise it starts collapsed.
- `<kc-tool>` renders nothing at all until `tool` is set — the `Show` guard means an empty element takes up no space.
- The element emits no events. When used inside `<kc-message>` or `<kc-chat>`, the enclosing element's `kc-message-action` event covers any parent-level interactions.
- The **Code** panel in the Preview above generates the exact framework snippet as you adjust the controls.
