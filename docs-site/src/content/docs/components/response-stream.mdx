<Aside type="tip" title="When to use">
Reach for `<kc-response-stream>` whenever you want to animate text as an assistant reply arrives — either replaying a finished string for visual effect, or driving the reveal in real time from an `AsyncIterable<string>`. The `typewriter` mode (default) is the right choice for most chat UIs; switch to `fade` for softer, document-style reveals. In SolidJS, use the `ResponseStream` primitive directly for finer control.
</Aside>

## Usage

Register the element once, then set the `text` **property** (not an attribute — async iterables can't be serialised to HTML attributes). Tune `mode` and `speed` as plain HTML attributes. Listen for `kc-complete` to know when the animation finishes.

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-response-stream id="stream" mode="typewriter" speed="20"></kc-response-stream>

<script type="module">
  const stream = document.getElementById('stream');
  // text can be a string, or an AsyncIterable<string> for live streaming
  stream.text = "Hello, this reveals one character at a time…";
  stream.addEventListener('kc-complete', () => console.log('done'));
</script>
```

Using ES module imports instead of the CDN bundle? Register the element with:

```js
import '@kitn.ai/chat/elements';
```

A few things worth knowing:

- **`text` is a JS property**, not an attribute. Assign it after the element upgrades — `el.text = '...'`. You can also pass an `AsyncIterable<string>` directly for a live-driven stream.
- **`mode`** is a plain attribute: `"typewriter"` (default) streams character by character; `"fade"` fades in segments.
- **`speed`** is a numeric attribute controlling characters (typewriter) or segments (fade) per tick. Default is `20`.
- **`as`** is an optional attribute to change the rendered wrapper element tag (e.g. `"p"`, `"span"`).
- **`kc-complete`** fires when the last character or segment has been revealed. See the **Events** table below for the detail shape, and the **Code** panel in the Preview above for per-framework listener syntax.
