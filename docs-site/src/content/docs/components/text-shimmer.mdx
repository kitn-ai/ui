<Aside type="tip" title="When to use">
Place `<kc-text-shimmer>` inline inside a status row, toolbar label, or above a streaming content area — anywhere you want to quietly communicate "working…" without a full spinner. It renders as an inline `span` so it flows naturally beside other text or icons. Swap it out for `<kc-thinking-bar>` when you also want a stop affordance.
</Aside>

## Usage

Register the custom elements once, then set the `text` attribute:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-text-shimmer text="Thinking…" duration="3" spread="20"></kc-text-shimmer>
```

In a bundled project, import the registration side-effect instead:

```ts
import '@kitn.ai/chat/elements';
```

All four props are **scalar attributes** — set them in HTML markup or via `setAttribute`. There are no JS-property-only props and no events to listen for.

**Tuning the effect** — `duration` controls how many seconds one sweep takes (default `4`; lower values are faster). `spread` controls the gradient width as a percentage (default `20`; range 5–45 — wider looks softer, narrower looks sharper).

**Rendering tag** — `as` lets you change the inner element from the default `span` to any inline or block tag (e.g. `"p"` or `"div"`) when the shimmer needs to participate in a different layout context.

### Fast sweep with wide spread

Lower `duration` to `2` and raise `spread` to `35` for a softer, quicker sweep — good for longer phrases like "Generating response…" where the slower default feels sluggish.

<Example client:visible tag="kc-text-shimmer" config={{ text: 'Generating response…', duration: '2', spread: '35' }} />

### Narrow, slow sweep

A narrow `spread` of `8` combined with the default duration creates a crisp, focused highlight — useful when the shimmer sits beside dense UI and needs to stay subtle.

<Example client:visible tag="kc-text-shimmer" config={{ text: 'Analysing…', spread: '8' }} />

### Rendered as a paragraph

Set `as="p"` to wrap the shimmer in a block-level element — handy when the label needs margin or sits as its own line above a streaming output area.
