<Aside type="tip" title="When to use">
Place `<kc-chain-of-thought>` above or below an assistant message to show the model's step-by-step plan or reasoning trace. It is `display: block` and sizes to content height, so no fixed height is required. In SolidJS, compose `ChainOfThought`, `ChainOfThoughtStep`, `ChainOfThoughtTrigger`, and `ChainOfThoughtContent` directly for finer control over animation and interactivity.
</Aside>

## Usage

Register the custom elements once, then hand the component its data:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-chain-of-thought id="cot"></kc-chain-of-thought>

<script type="module">
  document.getElementById('cot').steps = [
    { label: 'Understand the request', content: 'The user wants composable web components.' },
    { label: 'Design the API', content: 'Route 1: variant + flags; rich data via properties.' },
    { label: 'Build & verify' },
  ];
</script>
```

In a bundled project, prefer the ES module import over the CDN script:

```js
import '@kitn.ai/chat/elements'; // registers all kc-* custom elements
```

**`steps` is a JS property**, not an HTML attribute — assign it with `el.steps = [...]`. Each item is a plain object with a required `label` string and an optional `content` string. Steps that include `content` render an expand/collapse control; steps without `content` show the label only.

**Declarative children** — as an alternative to the property API, you can author each step as a `<kc-step>` child element in light DOM:

```html
<kc-chain-of-thought>
  <kc-step label="Understand the request">The user wants composable web components.</kc-step>
  <kc-step label="Design the API">Route 1: variant + flags; rich data via properties.</kc-step>
  <kc-step label="Build &amp; verify"></kc-step>
</kc-chain-of-thought>
```

The `label` attribute becomes the step heading; `textContent` becomes the optional expandable detail. The `<kc-step>` elements are invisible data carriers — the Shadow DOM suppresses them visually. You can also mix both APIs: `steps` property items render first, then any `<kc-step>` children are appended after.
