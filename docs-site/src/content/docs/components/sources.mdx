<Aside type="tip" title="When to use">
Place `<kc-sources>` directly below an assistant message when the reply cites external references. Use `numbered` when the answer body uses `[1]`, `[2]`, … citation markers. Use the `sources` property for dynamically constructed lists (streaming results, post-processing); use declarative `<kc-source>` children when the markup is static or server-rendered.
</Aside>

## Usage

Register the custom elements once, then set the `sources` **JS property** on the element:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<kc-sources id="srcs" show-favicon></kc-sources>

<script type="module">
  import '@kitn.ai/chat/elements';

  const srcs = document.getElementById('srcs');
  srcs.sources = [
    { href: 'https://kitn.dev', title: 'kitn — the kit', description: 'Composable chat UI.' },
    { href: 'https://solidjs.com', title: 'SolidJS', description: 'A reactive UI library.' },
  ];
</script>
```

A few things worth knowing:

- **`sources` is a JS property**, not an HTML attribute. Assign it with `el.sources = [...]`. Each item accepts `href` (required), `title`, `description`, `label`, and `showFavicon`.
- **`show-favicon`** is a boolean HTML attribute (or `el.showFavicon = true`). It enables favicons globally for all chips; a per-item `showFavicon: false` overrides it for that chip.
- **`numbered`** is a boolean attribute that replaces each chip's domain label (or custom `label`) with its 1-based index (`[1]`, `[2]`, …). Useful when the answer body references `[1]` in-line.
- **Declarative children** — you can skip the JS property entirely and write `<kc-source href="…" headline="…" description="…">` elements as light-DOM children of `<kc-sources>`. They are invisible data carriers (no shadow slot) picked up by a `MutationObserver`. Prop sources come first; declarative children are appended after. This is ideal for static or server-rendered markup where no JS wiring is needed.
- `kc-sources` emits no custom events of its own. The hover-cards are display-only.

For the exact per-framework wiring, see the **Code** panel in the Preview above.
