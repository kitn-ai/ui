<Aside type="tip" title="When to use">
Reach for `<kc-scroll-button>` whenever you render a long scrollable list outside SolidJS and want a "jump to latest" affordance — a chat thread, a log pane, an activity feed. Position it as an overlay (absolute/fixed) in the bottom-right corner of the scroll area. In SolidJS, the `ScrollButton` primitive inside a `ChatContainerRoot` does the same job without the web component wrapper.
</Aside>

## Usage

Register the custom elements once, then point the button at its scroll container using the `for` attribute:

```html
<script type="module" src="https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js"></script>

<div id="my-feed" style="height:400px; overflow-y:auto; position:relative;">
  <!-- list content -->
</div>

<!-- Overlay positioned in the bottom-right of the scroll area -->
<kc-scroll-button for="my-feed"></kc-scroll-button>
```

Or import from the npm package so the element registers as part of your bundle:

```js
import '@kitn.ai/chat/elements';
```

**All three props are scalar attributes** — set them directly in HTML or as attributes in your framework template. There are no JS-property data objects to assign.

**Wiring the scroll target** — use `for` with the DOM `id` of the scrollable container (mirrors the `<label for="...">` convention). When `for` is omitted the element walks up its own composed tree to the nearest scrollable ancestor — useful when `<kc-scroll-button>` is nested or slotted inside the container itself.

**Visibility behaviour** — the button is hidden when the container is within 50 px of the bottom and visible otherwise. The transition is animated (translate + scale + opacity). No polling: it listens to the container's `scroll` event passively.

**Listening for events** — the element fires `kc-scroll` (no detail payload) each time the button is clicked and `scrollToBottom()` completes. Use it to react to manual scroll events — for example, to pause auto-scrolling logic. See the **Events** table below and the **Code** panel in the Preview for the exact listener syntax in every framework.

### Ghost variant

The `ghost` variant removes the border and background for a more subtle overlay that sits unobtrusively over light content.

<Example client:visible tag="kc-scroll-button" config={{ variant: 'ghost' }} />

### Solid (default) variant

The `default` variant uses the filled button style from the design system — highest contrast, good for dark or image backgrounds where outline and ghost can get lost.

<Example client:visible tag="kc-scroll-button" config={{ variant: 'default' }} />

Use `size="icon-sm"` to produce a tighter button — fits well in compact panels or when the scroll area is narrow.
