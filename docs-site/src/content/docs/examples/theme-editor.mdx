## How it works

The controls don't touch component internals. They set inherited CSS custom properties on the host element, and the kit reads them through the Shadow DOM boundary. Editing live is one `setProperty` call per change:

```html
<kc-chat id="chat" chat-title="Brand assistant"></kc-chat>

<script type="module">
  import '@kitn.ai/chat/elements';

  const chat = document.getElementById('chat');

  // Seed the palette.
  const palette = {
    '--kc-color-primary': '#7c3aed',
    '--kc-color-primary-foreground': '#ffffff',
    '--kc-color-ring': '#7c3aed',
    '--kc-color-background': '#faf8ff',
    '--kc-color-card': '#ffffff',
    '--kc-color-border': '#e6deff',
    '--kc-radius': '0.625rem',
  };
  for (const [token, value] of Object.entries(palette)) {
    chat.style.setProperty(token, value);
  }

  // A color input re-skins the running chat on every change — no remount.
  primaryInput.addEventListener('input', (e) => {
    chat.style.setProperty('--kc-color-primary', e.target.value);
  });
</script>
```

The editor exposes the five tokens that carry the most visual weight, plus the corner radius:

| Token | Controls |
|---|---|
| `--kc-color-primary` | Buttons, links, the send action, active states |
| `--kc-color-ring` | Keyboard-focus outline — keep it high-contrast |
| `--kc-color-background` | The chat surface |
| `--kc-color-card` | Message bubbles and panels |
| `--kc-color-border` | Dividers and input outlines |
| `--kc-radius` | Corner rounding; `--radius-sm…xl` derive from it |

Set `--kc-color-primary-foreground` alongside `--kc-color-primary` so text stays legible on your accent. To ship a theme, drop the same block into a stylesheet — on the `kc-chat` selector to scope it, or on `:root` to rebrand every `kc-*` component at once.
