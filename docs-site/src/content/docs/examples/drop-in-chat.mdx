## Wire it up

Set `messages` as a JS **property**, then handle `kc-submit`: append the user's turn, add an empty assistant message, and grow its `content` as tokens arrive.

```html
<kc-chat id="chat" chat-title="Assistant" placeholder="Ask anything…"></kc-chat>

<script type="module">
  import '@kitn.ai/chat/elements';
  const chat = document.getElementById('chat');
  chat.messages = [];

  chat.addEventListener('kc-submit', async (e) => {
    const prompt = e.detail.value;
    const aId = crypto.randomUUID();
    // append the user turn + an empty assistant message
    chat.messages = [
      ...chat.messages,
      { id: crypto.randomUUID(), role: 'user', content: prompt },
      { id: aId, role: 'assistant', content: '' },
    ];
    chat.loading = true;

    // stream tokens from your backend into the assistant message
    for await (const token of streamFromYourModel(prompt)) {
      chat.messages = chat.messages.map((m) =>
        m.id === aId ? { ...m, content: m.content + token } : m,
      );
    }
    chat.loading = false;
  });
</script>
```

The key move is reassigning `chat.messages` to a **new array** on each token — that's what drives the re-render.
