## How it works

The `--kc-color-*` tokens are **inherited** CSS custom properties. Set them on the `<kc-chat>` element (or any ancestor) and they propagate into the Shadow DOM automatically — no stylesheet import required.

```html
<kc-chat id="chat" chat-title="Indigo Assistant"></kc-chat>

<script type="module">
  import '@kitn.ai/chat/elements';

  const chat = document.getElementById('chat');

  // Scope the brand tokens to this element only.
  chat.style.setProperty('--kc-color-primary',            'hsl(243 75% 59%)');
  chat.style.setProperty('--kc-color-primary-foreground', 'hsl(0 0% 100%)');
  chat.style.setProperty('--kc-color-ring',               'hsl(243 75% 65%)');
  chat.style.setProperty('--kc-color-background',         'hsl(244 20% 97%)');
  chat.style.setProperty('--kc-color-border',             'hsl(243 20% 88%)');
  chat.style.setProperty('--kc-radius',                   '0.5rem');

  // Wire messages + kc-submit as usual.
  chat.messages = [];
  chat.addEventListener('kc-submit', (e) => { /* stream reply */ });
</script>
```

To cover both modes in a stylesheet, write separate blocks for light and dark:

```css
/* light */
kc-chat {
  --kc-color-primary:            hsl(243 75% 59%);
  --kc-color-primary-foreground: hsl(0 0% 100%);
  --kc-color-ring:               hsl(243 75% 65%);
  --kc-color-background:         hsl(244 20% 97%);
  --kc-color-border:             hsl(243 20% 88%);
  --kc-radius:                   0.5rem;
}

/* dark — bump primary slightly for contrast on dark surfaces */
.dark kc-chat {
  --kc-color-primary:    hsl(243 75% 70%);
  --kc-color-ring:       hsl(243 75% 72%);
  --kc-color-background: hsl(243 20% 10%);
  --kc-color-border:     hsl(243 15% 20%);
}
```
