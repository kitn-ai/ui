Drop `<kc-chat>` into a page, set `messages` as a JS property, listen for `kc-submit`, and you have a full chat interface — thread, prompt input, loading state, and markdown rendering included.

## Install

```bash
npm install @kitn.ai/chat
```

Register the web components once at your app's entry point:

```js
import '@kitn.ai/chat/elements';
```

<Aside type="tip" title="No bundler?">
Load directly from a CDN — no install needed:

```html
<script type="module">
  import 'https://cdn.jsdelivr.net/npm/@kitn.ai/chat/dist/kitn-chat.es.js';
</script>
```

Pin a version in production (e.g. `@kitn.ai/chat@0.13.0/dist/kitn-chat.es.js`) — this package is pre-1.0 and minor versions may contain breaking changes.
</Aside>

## The minimal chat

`<kc-chat>` renders the full thread and prompt input. Set **`messages`** as a JS property (not an HTML attribute — arrays must go through the DOM property), then listen for **`kc-submit`** to handle user input.

```html
<body style="height: 100vh; margin: 0;">
  <kc-chat id="chat" style="display: block; height: 100%;"></kc-chat>

  <script type="module">
    import '@kitn.ai/chat/elements';

    const chat = document.getElementById('chat');

    chat.messages = [
      { id: '1', role: 'assistant', content: 'Hello! How can I help you today?' },
    ];

    chat.addEventListener('kc-submit', (e) => {
      console.log('User sent:', e.detail.value);
      // Call your model here
    });
  </script>
</body>
```

The `kc-submit` event fires with `{ value: string; attachments: AttachmentData[] }` in `e.detail`.

## Add streaming responses

On `kc-submit`, append the user message and an empty assistant message, then grow the assistant's content as tokens arrive. Reassign `chat.messages` with a **new array containing a new object** on each chunk — mutating in place won't trigger a re-render.

```js
import '@kitn.ai/chat/elements';

const chat = document.getElementById('chat');
chat.messages = [];

chat.addEventListener('kc-submit', async (e) => {
  const text = e.detail.value.trim();
  if (!text) return;

  // 1. Append the user turn
  const history = [
    ...chat.messages,
    { id: crypto.randomUUID(), role: 'user', content: text },
  ];
  chat.messages = history;
  chat.loading = true;

  // 2. Reserve a slot for the assistant reply
  const assistantId = crypto.randomUUID();
  chat.messages = [
    ...history,
    { id: assistantId, role: 'assistant', content: '' },
  ];

  try {
    // Point this at your own backend proxy — never expose the API key in the browser.
    const res = await fetch('/api/chat', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        messages: history.map((m) => ({ role: m.role, content: m.content })),
      }),
    });

    const reader = res.body.getReader();
    const decoder = new TextDecoder();
    let buffer = '';
    let answer = '';

    while (true) {
      const { value, done } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });

      const lines = buffer.split('\n');
      buffer = lines.pop(); // hold the partial last line
      for (const line of lines) {
        const s = line.trim();
        if (!s.startsWith('data:')) continue;
        const payload = s.slice(5).trim();
        if (payload === '[DONE]') continue;
        try {
          const delta = JSON.parse(payload).choices?.[0]?.delta?.content;
          if (!delta) continue;
          answer += delta;
          // New array + new object on every chunk
          chat.messages = chat.messages.map((m) =>
            m.id === assistantId ? { ...m, content: answer } : m
          );
        } catch { /* skip non-JSON keep-alive lines */ }
      }
    }
  } catch (err) {
    chat.messages = chat.messages.map((m) =>
      m.id === assistantId ? { ...m, content: `Error: ${err.message}` } : m
    );
  } finally {
    chat.loading = false;
  }
});
```

<Aside type="caution" title="Keep API keys on the server">
Never send a provider API key to the browser. Route `fetch` through your own backend endpoint that injects the key — the SSE parsing above is identical whether you proxy OpenRouter, Anthropic, or any OpenAI-compatible API.
</Aside>

## Framework wiring

The `kc-*` web components work in any framework. Set array/object data through the DOM property (not the HTML attribute), and listen for `kc-*` CustomEvents directly on the element.

    { id: '1', role: 'assistant' as const, content: 'Hello! How can I help?' },

      messages={messages}
      loading={loading}
      onSubmit={handleSubmit}
      style={{ display: 'block', height: '100vh' }}

  );
}

## ChatMessage shape

Each entry in `messages` is a `ChatMessage` object. The required fields are `id`, `role`, and `content`. The rest are optional enrichments:

```ts
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;

  // optional enrichments
  reasoning?: { text: string; label?: string };
  tools?: {
    type: string;
    state: 'input-streaming' | 'input-available' | 'output-available' | 'output-error';
    input?: Record<string, unknown>;
    output?: Record<string, unknown>;
    toolCallId?: string;
    errorText?: string;
  }[];
  attachments?: {
    id: string;
    type: 'file' | 'source-document';
    filename?: string;
    mediaType?: string;
    url?: string;
    title?: string;
  }[];
  actions?: ('copy' | 'like' | 'dislike' | 'regenerate' | 'edit' | {
    id: string; label: string; icon?: string; tooltip?: string;
  })[];
  avatar?: { src?: string; fallback?: string; alt?: string };
}
```

## Key props

| Prop | Type | Description |
|---|---|---|
| `messages` | `ChatMessage[]` | The full thread, newest last. **JS property only.** |
| `loading` | `boolean` | Shows streaming state; disables submit. |
| `placeholder` | `string` | Input placeholder text. |
| `suggestions` | `string[]` | Starter prompts shown on an empty thread. **JS property only.** |
| `suggestionMode` | `'submit' \| 'fill'` | `'submit'` sends on click; `'fill'` places in input. |
| `proseSize` | `'xs' \| 'sm' \| 'base' \| 'lg'` | Markdown body font scale. |
| `codeHighlight` | `boolean` | Set `false` to disable Shiki entirely. |

## Key events

| Event | `e.detail` | When |
|---|---|---|
| `kc-submit` | `{ value: string; attachments: AttachmentData[] }` | User sends a message. |
| `kc-value-change` | `{ value: string }` | Input text changes. |
| `kc-message-action` | `{ messageId: string; action: string }` | A message action button is clicked. |
| `kc-model-change` | `{ modelId: string }` | The model switcher changes. |

## Starter suggestions

Pass an array of `suggestions` to show prompt chips on an empty thread. Use `suggestionMode="fill"` to place the suggestion in the input instead of sending immediately.

```js
chat.suggestions = [
  'Summarize the latest news',
  'Write a Python function to parse CSV',
  'Explain how transformers work',
];
```

## Add syntax highlighting for more languages

Shiki loads on demand — only when a fenced code block actually renders. The default languages are `js`, `ts`, `tsx`, `json`, and `bash`. Add more:

```js
import { configureCodeHighlighting } from '@kitn.ai/chat/elements';

configureCodeHighlighting({
  languages: {
    python: () => import('@shikijs/langs/python'),
    rust:   () => import('@shikijs/langs/rust'),
  },
});
```

Disable highlighting entirely if you don't need it:

```js
configureCodeHighlighting({ enabled: false });
// or per element:
chat.setAttribute('code-highlight', 'false');
```

## Rebrand with CSS tokens

`kc-*` elements live in Shadow DOM, but they inherit CSS custom properties — override them on `:root` to rebrand without touching component internals.

```css
/* theme.css or your global styles */
:root {
  --color-background: #0f0f0f;
  --color-primary: #7c3aed;
  --color-muted: #1e1e1e;
}
```

For bundler projects you can also import the kit's token file:

```js
import '@kitn.ai/chat/theme.css';
```
